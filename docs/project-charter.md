# Vision

Project Fawkes is the restoration and modernization of a 2008 Apple iMac into a Linux-based engineering workstation.

The project is intended to preserve capable hardware while providing a platform to strengthen Linux administration, networking, automation, scripting, documentation, and security engineering skills.

Rather than simply installing a new operating system, Project Fawkes will document the complete engineering lifecycle from assessment through deployment and long-term operation.

---

# Mission

Restore.

Learn.

Build.

Share.

---

# Objectives

- Extend the useful life of legacy hardware.
- Install Linux Mint as the primary operating system.
- Build a workstation suitable for MSP engineering.
- Create repeatable engineering documentation.
- Develop Linux administration skills.
- Learn Bash scripting and automation.
- Build a searchable engineering knowledge base.
- Publish selected work to a personal website.

---

# Scope

## Included

- Hardware restoration
- SSD installation
- Linux Mint deployment
- Driver validation
- Performance benchmarking
- Engineering software installation
- Documentation
- Automation
- Security hardening
- Personal portfolio development

## Excluded

- Hardware overclocking
- Unsupported hardware modifications
- Replacement of working components without a clear benefit

---

# Success Criteria

Project Fawkes will be considered successful when:

- Linux Mint operates reliably.
- All major hardware functions correctly.
- The workstation is used regularly for engineering tasks.
- Documentation is complete and maintained.
- The project is published as part of a professional portfolio.

---

# Engineering Principles

1. Document before changing.
2. Measure before optimizing.
3. Preserve original hardware whenever practical.
4. Favor open standards.
5. Build repeatable processes.
6. Share knowledge.

---

# Long-Term Vision

Project Fawkes becomes the first volume of a personal engineering portfolio that demonstrates practical skills through real-world projects rather than certifications alone.
